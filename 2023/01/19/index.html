HK365找工周记1

24年七月毕业，希望能找到理想的工作！！！

先交代一下自己的背景：

211软件工程本，港三MEng。自认为编码能力一般。各种语言都学过一点，Java, Python, C, C++, Solidity, H5+CSS，Js。基本都会用但也仅限于会用，都只有用来搭建小项目的经验。目前能用来作为主力工作语言的也就只有Java，用springboot+vue这一技术栈做过一些项目。

目标公司：

  1. HK 银行、券商、投行的IT Support部门SDE岗等
  2. 潜力初创Web3公司, backend开发或者 backend+solidity开发。大型虚拟货币交易所SDE，eg: OKX python SDE， OKX JAVA SDE
  3. Technology Consulting，eg: 四大，IBM
  4. HK 外资互联网公司 JAVA的SDE以及Fintech开发
  5. 内地外资互联网大厂/中资互联网大厂养老组

一些很感兴趣并且想了解的工作（不排斥转行）：

  1. 高频交易量化策略
  2. 金融科技行研以及互联网企业行研

对香港IT的一些认知：香港的互联网发展远远不如隔壁深圳，IT业几乎所有的SDE岗位都是为金融所服务的，本土互联网企业很少且Junior SDE跟洗碗工差不多工资，技术也比较陈旧，稍微翻一下jdb, linkdIn, glassdoor, CTjob就会发现，很多公司现在还在用node，引入Java和Go的都不多（没有说node不好的意思）。学区块链的话，有前景，但是打开jdb，你会发现只有不到10个岗位，而且几乎没有junior岗位, 所以只能考虑科技园的一些小型初创企业以及做remote。在算法岗方面，主流的就是NLP以及CV了吧，岗位极少，基本也就华为港研所和商汤招人多一点。Data方面的岗位我不太感兴趣,但是挺多的。

找行研倒是出路，行业发展也好。但是，一方面感觉大学四年加研究生都白读了。另一方面，觉得还是有行业壁垒，贸然转行还是存在风险的，毕竟人很难赚到自己认知以外的钱。自己在学习区块链知识的时候也会去了解一些Defi，ESG等，也在binance、okx、huobi一些平台上做现货交易，还上了Financial computing的课程，但觉得自己对它们的认知还是非常浅薄，有在考虑要不要考CFA一级试试水。
